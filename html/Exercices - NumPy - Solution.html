La rgression linaire simple fait l'hypothse que la relation entre la variable d'intrt et la variable en entre est modlise par une relation linaire :

$$ y \approx a x + b $$

Pour estimer le coefficient $a$ et la constante $b$, on utilise le critre de la somme des moindres carrs :

$$ \hat{a}, \hat{b} = \arg\min_{a, b} \sum_{i=1}^n \left( y^{(i)} - (a x^{(i)} + b) \right)^2 $$

Les solutions ce problme d'optimisation (obtenues en cherchant quand le gradient de cette fonction est nul) sont les suivantes :

$$ \hat{a} = \frac{\text{Cov}(x, y)}{\text{Var}(x)} = \frac{\sum_{i=1}^n \left( x^{(i)} - \bar{x} \right) \left( y^{(i)} - \bar{y} \right)}{\sum_{i=1}^n \left( x^{(i)} - \bar{x} \right)^2} \qquad\text{et}\qquad \hat{b} = \bar{y} - \hat{a} \bar{x} $$

avec :

$$ \bar{x} = \frac{1}{n} \sum_{i=1}^n x^{(i)} \qquad\text{et}\qquad \bar{y} = \frac{1}{n} \sum_{i=1}^n y^{(i)} $$

Rgression linaire multiple

Passons maintenant un cas plus complexe, mais plus raliste, avec la rgression linaire multiple. En effet, il est irraliste dans l'immense majorit des cas d'esprer pouvoir prdire la variable d'intrt $y$ avec une seule variable en entre.

On suppose maintenant que chaque entre n'est plus un rel $x^{(i)} \in \mathbb{R}$ mais un vecteur de nombre rels $\mathbf{x}^{(i)} \in \mathbb{R}^p$. chaque entre $\mathbf{x}_i$ est toujours associe une sortie $y^{(i)} \in \mathbb{R}$. Les entres sont reprsentes par la matrice $\mathbf{X} \in \mathbb{R}^{n \times d}$ et les sorties sont reprsentes par le vecteur $\mathbf{y} \in \mathbb{R}^{n}$.

On utilise nouveau la fonction sklearn.datasets.make_regression() pour gnrer un tel jeu de donnes :

Deux variables Python ont t cres :

  • X_lin correspond la matrice $\mathbf{X}$ contenant les entres,
  • y_lin correspond au vecteur $\mathbf{y}$ contenant les sorties.

La rgression linaire multiple fait l'hypothse que la relation entre la variable d'intrt et les variables en entre est modlise par une relation linaire :

$$ y \approx w_0 + \sum_{j=1}^p w_i x_i $$

Une manire courante de rajouter la constante est de rajouter une variable supplmentaire qui vaut 1, c'est--dire une colonne de 1 la matrice $\mathbf{X}$ :

$$ y \approx \sum_{j=0}^p w_i x_i = \mathbf{w}^\top \mathbf{x} $$

On utilise toujours le critre de la somme des moindres carrs pour trouver les paramtres optimaux :

$$ \hat{\mathbf{w}} = \arg\min_{\mathbf{w}} \sum_{i=1}^n \left( y^{(i)} - \mathbf{w}^\top \mathbf{x}_i \right)^2 = \arg\min_{\mathbf{w}} \Vert \mathbf{y} - \mathbf{X} \mathbf{w} \Vert_2^2 $$

On calcule le gradient de cette fonction par rapport $\mathbf{w}$ et on cherche quand il est nul :

$$ -2 \mathbf{X}^\top \left( \mathbf{y} - \mathbf{X} \hat{\mathbf{w}} \right) = 0 \Longleftrightarrow \mathbf{X}^\top \left( \mathbf{y} - \mathbf{X} \hat{\mathbf{w}} \right) = 0 \Longleftrightarrow \mathbf{X}^\top \mathbf{X} \hat{\mathbf{w}} = \mathbf{X}^\top \mathbf{y} $$

On obtient donc l'quation suivante :

$$ \mathbf{X}^\top \mathbf{X} \hat{\mathbf{w}} = \mathbf{X}^\top \mathbf{y} $$

Vous avez peut-tre envie d'inverser la matrice $\mathbf{X}^\top \mathbf{X}$ pour trouver $\hat{\mathbf{w}}$, mais ce n'est pas une bonne ide. En effet, on n'a pas besoin de $\left(\mathbf{X}^\top \mathbf{X}\right)^{-1}$, qui est une matrice de taille $(p + 1) \times (p + 1)$, mais seulement de $\left( \mathbf{X}^\top \mathbf{X} \right)^{-1} \mathbf{X}^\top \mathbf{y}$, qui est un vecteur de taille $p + 1$. C'est pourquoi on laisse l'quation sous la forme d'un systme d'quations linaires, parce qu'il est plus efficace de la rsoudre sous cette forme :

$$ \mathbf{X}^\top \mathbf{X} \hat{\mathbf{w}} = \mathbf{X}^\top \mathbf{y} $$

Rgression logistique

La rgression logistique est un modle linaire pour des tches de classification binaire. L'objectif n'est pas de prdire un nombre rel mais une classe parmi deux choix possibles (par exemple malade ou sain dans le domaine mdical). L'algorithme s'tend aux tches de classification multiclasse, c'est--dire avec un nombre de classes strictement plus grand que deux, mais nous nous limiterons la classification binaire ici.

Un modle linaire de classification binaire spare l'espace en deux sous-espaces par un hyperplan. En classification binaire, on fait souvent rfrence aux deux classes par les termes classe positive et classe ngative.

Le modle prdit comme :

  • observation positive tout point du plan dont la distance signe l'hyperplan est positive, et
  • observation ngative tout point du plan dont la distance signe l'hyperplan est ngative.

Pour un hyperplan caractris par le vecteur $\mathbf{w}$, la distance signe d'un point $\mathbf{x}$ l'hyperplan est le produit scalaire entre les deux vecteurs :

$$ f(\mathbf{x}; \mathbf{w}) = \mathbf{w}^\top \mathbf{x} $$

On fait encore l'hypothse d'avoir rajouter une variable supplmentaire au vecteur $\mathbf{x}$ contenant un 1 pour modliser la constante.

La rgression logistique est un modle probabiliste avec l'hypothsse suivante :

$$ P(y=1 \vert \mathbf{x}) = \sigma \left( \mathbf{w}^\top \mathbf{x} \right) $$

o $\sigma$ est la fonction sigmode dfinie par :

$$ \sigma(x) = \frac{1}{1 + \exp(-x)} $$

def sigmoid(x):
    return 1 / (1 + np.exp(-x))

Pour trouver les paramtres optimaux (l'hyperplan $\mathbf{w}$), on cherche encore minimiser une fonction de cot note $J$ :

$$ \mathbf{w}^* = \arg\min_{\mathbf{w}} J(\mathbf{w}) $$

On utilise l'entropie croise comme critre :

$$ J(\mathbf{w}) = \frac{1}{n} \sum_{i=1}^n - y^{(i)} \log \left( \sigma \left( \mathbf{w}^\top \mathbf{x}^{(i)} \right) \right) - \left( 1 - y^{(i)} \right) \log \left( 1 - \sigma \left( \mathbf{w}^\top \mathbf{x}^{(i)} \right) \right) $$

Le gradient de cette fonction par rapport $\mathbf{w}$ est not $\nabla_{\mathbf{w}} J$ :

$$ \nabla_{\mathbf{w}} J(\mathbf{w}) = \frac{1}{n} \sum_{i=1}^n \left( \sigma \left( \mathbf{w}^\top \mathbf{x}^{(i)} \right) - y^{(i)} \right) \mathbf{x}^{(i)} $$

Il n'existe pas d'expression de forme ferme pour rsoudre l'quation $\nabla_{\mathbf{w}} J(\mathbf{w}) = 0$. On va donc implmenter la place une descente du gradient pour trouver les paramtres optimaux.

Algorithme de descente du gradient

  • Paramtres :
    • X : matrice des entres
    • y : vecteur des sorties
    • max_iter : nombre maximum d'itrations
    • tol : tolerance (sur la norme infinie du gradient)
    • lr : taux d'apprentissage $\eta$
  • Initialiser les coefficients 0 : $\mathbf{w}^{(0)}$
  • Calculer le gradient $\nabla_{\mathbf{w}} J(\mathbf{w}^{(0)})$
  • Si la norme infinie du gradient est infrieure tol
    • Renvoyer les rsultats : l'initialisation convient
  • Sinon, tant que le nombre maximum d'itrations n'est pas atteint
    • Mettre jour les coefficients avec la formule suivante : $\mathbf{w}^{(t)} = \mathbf{w}^{(t-1)} - \eta \times \nabla_{\mathbf{w}} J(\mathbf{w}^{(t-1)})$
    • Calculer le gradient $\nabla_{\mathbf{w}} J(\mathbf{w}^{(t)})$
    • Si la norme infinie du gradient est infrieure tol
      • Arrter
  • Renvoyer les coefficients finaux, si l'algorithme a converg et le nombre d'itrations effectues.

Pour rappel, la norme infinie d'un vecteur est dfinie par : $$ \Vert \mathbf{x} \Vert_{\infty} = \max_{j} \vert x_j \vert $$

def function_logistic_regression(w, X, y):
    y_proba = sigmoid(X @ w)
    return - np.mean(y * np.log(y_proba) + (1 - y) * np.log(1 - y_proba))

def gradient_descent(X, y, max_iter, tol, lr):

    # Initialisation des coefficients
    w = np.zeros(X.shape[1])
    
    # Calcul du gradient
    grad = gradient_logistic_regression(w, X, y)

    # Si la norme infinie du gradient est infrieure  tol
    if norm(grad) < tol:

        # Renvoyer les rsultats
        return {'coef': w, 'converg': True, 'n_iter': 0}
    
    # Initialisation de la convergence
    converge = False

    # Tant que le nombre maximum d'itrations n'est pas atteint
    for n_iter in range(1, max_iter + 1):
        
        # Mettre  jour les coefficients
        w -= lr * grad

        # Calculer le gradient
        grad = gradient_logistic_regression(w, X, y)

        # Si la norme infinie du gradient est infrieure  tol
        if norm(grad) < tol:

            # Il y a convergence : arrter l'algorithme
            converge = True
            break

    # Renvoyer les rsultats
    return {'coef': w, 'converg': converge, 'n_iter': n_iter}

Il existe une borne suprieure qui permet d'affirmer que si le taux d'apprentissage est infrieur cette borne suprieure, alors l'algorithme converge forcment (avec assez d'itrations) vers la solution optimale. Cette borne suprieure est l'inverse de la plus grande valeur propre de la matrice hessienne de la fonction de cot.

La matrice hessienne de la fonction de cot est : $$ \nabla_{\mathbf{w}}^2 J(\mathbf{w}) = \frac{1}{n} \sum_{i=1}^n \sigma \left( \mathbf{w}^\top \mathbf{x}^{(i)} \right) \left( 1 - \sigma \left( \mathbf{w}^\top \mathbf{x}^{(i)} \right) \right) \mathbf{x}^{(i)} \mathbf{x}^{(i)\top} = \frac{1}{n} \mathbf{X}^\top \mathbf{D} \mathbf{X} \qquad\text{avec}\qquad \mathbf{D} = \text{diag}\left( \left[ \sigma \left( \mathbf{w}^\top \mathbf{x}^{(i)} \right) \left( 1 - \sigma \left( \mathbf{w}^\top \mathbf{x}^{(i)} \right) \right) \right]_{i=1}^p \right) $$

La matrice hessienne dpend du point d'valuation $\mathbf{w}$ travers la matrice diagonale $\mathbf{D}$. Nanmoins, si on trouve un majorant de $\mathbf{D}$ (en valeurs propres), alors on pourra calculer un majorant de la hessienne.

Question 26 : Dfinissez une nouvelle version de la fonction gradient_descent_auto_lr() qui ne prend plus un argument lr pour le taux d'apprentissage, mais qui le calcule automatiquement avec la mthodologie dfinie ci-dessus. In[33]:
def gradient_descent_auto_lr(X, y, max_iter, tol):

    # Initialisation des coefficients
    w = np.zeros(X.shape[1])

    # Calcul du gradient
    grad = gradient_logistic_regression(w, X, y)

    # Si la norme infinie du gradient est infrieure  tol
    if norm(grad) < tol:

        # Renvoyer les rsultats
        return {'coef': w, 'converg': True, 'n_iter': 0, 'lr': None}

    # Calculer le taux d'apprentissage optimal
    lr = (4 * X.shape[0]) / svds(
        X_log, k=1, which='LM', rng=42, return_singular_vectors=False
    ).item() ** 2

    # Initialisation de la convergence
    converge = False

    # Tant que le nombre maximum d'itrations n'est pas atteint
    for n_iter in range(1, max_iter + 1):

        # Mettre  jour les coefficients
        w -= lr * grad

        # Calculer le gradient
        grad = gradient_logistic_regression(w, X, y)

        # Si la norme infinie du gradient est infrieure  tol
        if norm(grad) < tol:

            # Il y a convergence : arrter l'algorithme
            converge = True
            break

    # Renvoyer les rsultats
    return {'coef': w, 'converg': converge, 'n_iter': n_iter, 'lr': lr}
